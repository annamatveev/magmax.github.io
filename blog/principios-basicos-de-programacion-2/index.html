Principios básicos de programación 2

2015-10-03

Miguel Ángel García

Hace tiempo que estoy pensando cómo continuar el artículo Principios básicos de programación, pero es más complicado de lo que parece.

¿Cómo continuar un tutorial de programación genérico, sin atarme a ningún lenguaje?

Si sólo quisiera dar teoría, comenzaría a clasificar los lenguajes de programación según distintas clasificaciones: proximidad al lenguaje máquina, uso, etc.

Sin embargo voy a hacer algo distinto. Voy a comentar las herramientas típicas de que se dispone si el lenguaje es estructurado, orientado a objetos o funcional.

Programación Estructurada

Cuando hablamos de programación Estructurada nos referimos a lenguajes como Basic, Pascal, C, ... Aunque hay otros lenguajes que son mixtos, como Python, Ruby, C++, ...

La característica principal de la programación estructurada es que el código tiene un orden. Lo habitual es que se ejecute siempre una instrucción tras otra, con las excepciones de las llamadas a funciones y la instrucción goto (a menudo prohibida por fomentar malas prácticas).

Un ejemplo podría ser la receta para hacer pizza: cada paso tiene un lugar, y no se puede alterar. Echar el tomate antes que el queso implica hacer una pizza diferente, sería una receta distinta.

Las ventajas de la programación estructurada son claras: es fácil de entender y el código es fácil de seguir.

Sin embargo, cuando se comienza a programar algo más grande se tiende a utilizar estructuras de datos más complejas y a tratar de reutilizar más código. Por ejemplo, si nos creamos una "cola de la pescadería" veremos que se parece mucho a la "cola de la carnicería", por lo que podemos ahorrar código.

Estas estructuras complejas están compuestas de datos y operaciones. Datos como las personas que forman la cola y operaciones tales como "pedir la vez" o "siguiente en la cola".

Una aproximación seguida en los lenguajes estructurados es utilizar TADs, donde se crea una estructura y todas sus acciones/funciones requieren un objeto de esa estructura como primer argumento. Ejemplo:

String Persona // una persona es sólo un nombre
String lista = new Array(Persona);


pedir_la_vez(lista, persona);
siguiente_en_la_cola(lista);
